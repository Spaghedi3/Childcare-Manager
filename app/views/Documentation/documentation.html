Software Requirements Specification (SRS)

Childcare Manager

1. Introduction

1.1 Purpose

The purpose of the Childcare Manager software is to assist parents, couples, and caretakers in managing childcare tasks and tracking important developmental milestones of children.

1.3 Product Scope

The Childcare Manager application will include features such as managing feeding and sleep schedules, tracking developmental milestones through multimedia resources (photos, videos, audio recordings), maintaining medical history, and facilitating social sharing of significant moments. The system will provide a chronological view of captured moments for each child, with options for social sharing via social applications or RSS news feeds. Additionally, the system functionalities will be exposed through a REST/GraphQL Web API.

2. Overall Description

2.1 Product Functions

The major functions of the Childcare Manager application include:

  • Feeding schedule management
  • Sleep schedule management
  • Multimedia resource management for child development tracking
  • Medical history recording
  • Relationship tracking with other children
  • Chronological view of important moments
  • Social sharing via social applications or RSS news feeds
  • Exposing functionalities through a REST/GraphQL Web API

2.2 User Classes and Characteristics

User classes for the Childcare Manager application include parents, couples, and caretakers. Users may vary in technical expertise and familiarity with childcare management software.

2.3 Operating Environment

The Childcare Manager application will operate on various platforms, including web browsers, mobile devices (iOS and Android), and desktop computers. It will require internet connectivity for social sharing features and accessing multimedia resources.

2.4 User Documentation

User documentation will include user manuals, on-line help, and tutorials. The delivery format will include web-based resources accessible through the application interface.

2.5 Assumptions and Dependencies

Assumptions:

  • Users will have access to internet connectivity for social sharing features.

Dependencies:

  • The application may depend on third-party services for social sharing functionality.

3. External Interface Requirements

3.1 User Interfaces

The user interface will be intuitive and user-friendly, featuring screens for feeding schedule management, sleep schedule management, multimedia resource management, medical history recording, relationship tracking, and social sharing.

3.2 Hardware Interfaces

The application will interface with standard hardware components of web browsers on mobile devices, and desktop computers.

3.3 Software Interfaces

The application will interface with third-party social applications for social sharing functionality. It will also expose functionalities through a REST/GraphQL Web API.

3.4 Communications Interfaces

Communication interfaces will include internet connectivity for social sharing features and accessing multimedia resources.

4. System Features

4.1 Feeding and sleeping Schedule Management

4.1.1 Description and Priority

The system allows users to record and manage feeding and sleeping schedules for children. Priority: High.

4.1.2 Stimulus/Response Sequences

Users input feeding and sleeping times. The system records and displays the feeding and sleeping schedule.

4.1.3 Functional Requirements

  • The system provides reminders for upcoming feedings and sleep times.

4.2 Gallery and timeline management

4.2.1 Description and Priority

The system allows users to add and manage images, videos and other media types of content for children. Priority: High

These media files can be shared on social media.

4.2.2 Stimulus/Response Sequences

Users input media files. The system records and displays them in the timeline and gallery.

The timeline will offer users a friendly interface for viewing, adding and deleting certain memories. Additionally these memories will be showcased in a chronological order.
